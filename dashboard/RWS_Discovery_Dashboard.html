Programme Rules 7 items
🛡️
Fraud Detection & Prevention
Real-time anomaly detection for member accounts & transactions. Customisable thresholds per store group (retail, F&B, ticketing, hotels) and spend channel. Alerts, push notifications, and pre-set suspension actions. Configurable dashboard with custom timespan and export.
✗ NOT IN BRD Programme Rules
💰
Points Earning Rules — Payment Mode Eligibility
Points earned on cash, credit, debit only. Certain stores award tier-qualifying points only (no spendable points). Eligible payment modes for points earning and tier eligibility must be configurable per store or channel.
⚠ PARTIAL — FR 9.1.6.01 covers earn rate only
🏪
New Store Creation — Extended Fields
Store fields required: RWS/3P flag · Store Name · Store Type · Store Location Group · OU/CC & GL information · Unique identifier for reports. Active/inactive store differentiation.
⚠ PARTIAL — FR 9.1.4.01 exists but GL/OU fields not specified
🔄
Void Transactions
Voided earn/redemption transactions reversed to original point source. All voids reflected in Finance reports. Same-day void restriction only.
✗ EXPLICITLY OUT OF SCOPE in BRD §7.03 ⚠ CONFLICT: BU needs this at launch
⚡
Fast Track Tier Progression & Partner Tier Match
Fast track members between non-adjacent tiers (e.g. A → C). Real-time tier upgrade/downgrade based on eligible spend. Tier match with partner programs (e.g. KrisFlyer). Partner fast-track with minimum spend at RWS or partner.
✗ NOT IN BRD Programme Rules
⛔
Member Suspension & Deletion (Two States)
Suspended: app content accessible, no transactions, cannot re-register with same credentials. Deleted: no app login, previous credentials released for re-use.
✗ NOT IN BRD — active_status (0/1/2) defined but logic incomplete
👤
Pre-Created Staff Accounts (Default Password Flow)
Pre-create provision for new staff accounts so staff download app and sign in with a default password — eliminating self-registration. Bulk employee account provisioning.
✗ NOT IN BRD Programme Rules

✏️
Member Profile Update via Concierge Portal

✓ IN BRD — FR 9.1.5.03 (edit profile via API)
➕
Member Account Creation via Concierge Portal
Concierge staff can create member accounts on behalf of members without requiring the member app.
⚠ PARTIAL — FR 9.1.5.01 covers API creation but no concierge portal UI specified
🏅
Tier Adjustment — Add TQP Without Affecting Expiry
Add tier-qualifying points (TQP) to a user's account for missing transactions without resetting the existing TQP expiry date.
⚠ PARTIAL — FR 9.1.5.19 covers tier adjust but TQP expiry preservation not specified

Campaigns 4 items
🔀
Mutually Exclusive / Either-Or Campaign Setup
Allow either/or campaign logic with mutual exclusion. Example: Spend $25 on single receipt OR spend $288 with 3 receipts — member can only qualify for one. Plus rollover quantity cap per day.
✗ NOT IN BRD — §9.1.8 has no mutual exclusion logic
🔢
Decimal Point Cap on Point Source Restriction
Allow decimal point values in campaign-level point source caps. Example: restrict a campaign to award max 0.5x point multiplier.
✗ NOT IN BRD — §9.1.7 has no decimal cap specification
🎟️
Promo Codes — Custom Prefix, Points Cap, Redemption Report
Custom promo code naming prefix (for points vs promo). Max points per user capped at campaign level for both unique and generic codes. View report of successful redemptions — currently unavailable.
✗ NOT IN BRD — no promo code requirements in §9.1.8 or §9.1.11
🔗
Referral Codes — Multi-criteria Referrer & Referee Rewards
Customised outcomes for referrer and referee on successful signup. Referrer gets points + vouchers when referee makes $X transaction or refers N users. Cannot currently layer multiple criteria simultaneously.
✗ NOT IN BRD — referral not mentioned in any section ⚠ Duplicate item — also listed in Referral category
