🔴 Critical Gaps — Block BRD Skill Compliance & Gate 1
🚨

5 Critical Gaps — Must Fix Before Gate 1 Sign-Off

1

Zero Acceptance Criteria on ALL ~80 Functional Requirements

The Skill mandates: "Every FR must have at least 1 acceptance criterion." The RWS BRD has 0 AC across all 16 functional sections. Without AC, UAT sign-off and go-live gate authorisation have no objective basis.

✅ Example fix — FR 9.1.5.01: AC1: Profile created in SessionM within 2s of API call · AC2: All mandatory custom attributes from §9.1.5.1 populated · AC3: Duplicate email returns 409 Conflict

2

Non-Functional Requirements — Only 2, No Measurable Targets, Blank Systems & Rating

Skill requires NFRs with numeric measurable targets. For 250 stores processing real-time loyalty transactions, this is a critical omission. Missing NFRs include performance, availability, DR, PDPA, and security.

✅ Minimum needed: API response ≤500ms p95 · Availability ≥99.9% · PDPA Singapore consent · DR: RTO ≤4h RPO ≤1h · Scalability: 10x member growth

3

No Priority Tags on Any FR (Must Have / Should Have / Nice to Have)

None of the ~80 FRs carry a priority. The Skill requires this on every requirement. Without priority data, the Jira export cannot assign High/Medium/Low to stories, and sprint planning has no prioritisation basis.

4

No System Architecture Overview Section

Skill Section 8 is entirely absent. No component diagram or description of how SessionM, RWS Middleware, Member App, Merchant App, Microsoft Entra, and AWS S3 Data Cloud connect. The Solution Architect cannot produce the SDD without this foundation.

✅ Minimum needed: System component list · Data flow directions · API integration points (sync/async) · Auth flow (Entra → SessionM) · Merchant App → Middleware → SessionM pathway

5

No Risk Register

Skill Section 12 is entirely absent. For an MVP loyalty launch across 250 stores with PDPA PII implications (car plates, IU numbers, mobile numbers), this is a Gate 1 blocker. Key undocumented risks: PDPA non-compliance, merchant onboarding delays, SessionM API reliability, campaign misconfiguration, and member data accuracy.

⚠ Warnings & Improvements
⚠️

Warnings & Nice-to-Have Improvements

W1

No User Personas

Stakeholders are listed but the Skill requires actual end-user personas. Missing: Tourist/Visitor, RWS Regular Guest, RWS Employee (TM), and Merchant Staff — each needs goals, pain points, and key interactions with the loyalty program.

W2

Business Requirements Not Structured as BR-XXX

The 5 objectives in §5 are business goals but not formatted as BR-XXX with priority and rationale. Conversion needed: e.g. BR-001 Customer Retention · Must Have · Rationale: reduce churn by 15%.

W3

No User Journey / Process Flows

Skill §10 requires step-by-step flows with happy path + error paths. All absent: Member Registration, Points Earn (Merchant App → SessionM), Points Redemption, Tier upgrade/downgrade, Offer issuance and redemption via voucher management system.

W4

No Phase Tagging on Any Requirement

BRD acknowledges MVP phasing in assumptions but no individual FR carries a Phase 1/2/3 tag. This makes sprint backlog prioritisation unreliable and the Jira export cannot set Fix Version/Sprint correctly.

W5

Milestones Section (§4) is Empty

Header present but no content. Needs: milestone table with target dates, phase criteria, and gate conditions (Gate 1, UAT start, go-live).

N1

Glossary Has Only 1 Term

Needs: xplore Dollar · Tier Qualification · Tier Maintenance · RLP_ID · RLP_NO · IU Number · Track Tier C · Wander/Discover/Journey/Pinnacle · NG Dollar · TM · NGLP · MVP · 3P · SMP

N2

FR Numbering Mismatch

Section 9.1.2 (Data Ingestion) uses requirement numbers 9.1.3.xx. This creates traceability errors across the BRD and will cause issues in the Jira ticket linking. All req numbers must align to their section numbers.

N3

No PDPA Singapore Compliance Requirement

The program collects significant PII: name, email, mobile, car plate, IU number. Singapore's PDPA obligations — consent collection, data access rights, right to deletion, and breach notification — are entirely absent from the BRD.

N4

No Executive Summary Section

Skill requires a structured 3–5 sentence exec summary with business problem, proposed solution, and quantified success metrics (target member count, retention rate improvement, revenue uplift target).

N5

NFR Rating & Systems Columns Blank

Both existing NFR rows have empty System and Rating columns. Quick fix: Rating = Must Have, Systems = SessionM. Then add the 5+ missing NFRs with measurable targets.
