Automated Tests Run

Code Review (2 approvals)

Merge → Auto Deploy to Staging

5️⃣

Release → Production

Manual gate → blue/green deploy → monitor

🔬 6 Required Test Types

1

Functional Testing

Validates system behaviour vs BRD FRs. Every FR → ≥1 positive TC.

2

Integration Testing

API contracts, data flows, 3rd-party systems, message queues.

3

Negative Testing

Invalid inputs, missing fields, auth failures, error conditions.

4

Edge Case Testing

Boundary values, concurrency, timezone handling, empty states.

5

Regression Testing

Existing functionality unaffected by new code — run after every fix.

6

UAT — User Acceptance

Business stakeholders validate real workflows in UAT environment.
